# Project: Oracle VM Creator

## Architecture
- **Script**: Python script using Playwright to automate the VM creation.
- **Base URL**: Configurable Oracle Cloud Console URL (defaults to real URL, but supports custom URL for testing).
- **Mock Console**: A local web server (e.g., Flask or http.server) simulating the Oracle Cloud login and instance creation flows.
- **Hybrid Pause**: Execution pause mechanism that waits for a user keypress/input in the terminal after displaying the login page.
- **SSH Keys**: Automatic retrieval/handling of SSH keys generated by Oracle Cloud Console, saving them to the output directory.

## Code Layout
- `create_vm.py`: The main automation script.
- `mock_console/`: Directory containing the mock server for testing.
  - `server.py`: Flask-based mock server.
  - `templates/`: HTML templates for login, dashboard, instance list, and instance creation.
- `tests/`: E2E and unit test suite.
  - `test_vm_creator.py`: Test cases using Playwright.
- `requirements.txt`: Python package dependencies (Playwright, Flask, etc.).

## Interface Contracts
### Playwright Script Interface
- Command: `python create_vm.py --url <console_url> --output-dir <dir>`
- Inputs:
  - `--url`: (Optional) URL of the Oracle Cloud Console or local mock.
  - `--output-dir`: (Optional) Directory to save SSH keys.
- Output:
  - Downloaded SSH private/public key files in the output directory.
  - Exit code 0 on success.
  - Informative terminal output during hybrid pause.

### Mock Server Interface
- Login Page: Simulates login and redirects to dashboard/instances list after form submission.
- Instances Page: Lists instances and contains a "Create Instance" button.
- Create Instance Page: Contains form fields for name (`speack3-server`), OS Image choice, Shape choice, and SSH key download button.
